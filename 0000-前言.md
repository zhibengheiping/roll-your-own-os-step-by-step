# 前言

一说起操作系统，很多人会觉得它再简单不过了——不就是让一台计算机同时跑多个程序嘛。可一旦你自己动手去写，哪怕只是迈出第一步，也常常会感到无从下手。

随着技术热点的不断更迭，操作系统早已沦为一个冷门方向。如今能找到的自己动手写操作系统的教程，大多还是二十多年前的老古董，讲的仍是串口终端、IDE接口的硬盘这些早已淡出主流的东西。计算机底层早已发生了天翻地覆的变化，就连图形界面底层现在也都是基于OpenGL的了。

如今的操作系统，早已被一层又一层的抽象封装得严严实实，底层真面目几乎辨认不出了。我们每天调用的接口都像是漂浮在海面上的冰山一角，而水下的，对很多即便是专业程序员来说，都是一团迷雾。

别慌，哪怕你现在只会写一个 `printf("Hello, world\n");` ，只要跟着本教程一步一步来，也能从零开始，手搓出一个真正能跑的操作系统。

你不用担心一开始就要去啃厚厚的硬件手册，也不用被迫去读其他操作系统的源码。我们首先会在Linux环境下直接调用底层接口来写程序。这些程序既帮你熟悉底层接口，后面也会拿来验证我们自己写的操作系统到底能不能正常运行起来。

现代Linux提供了丰富的接口。在用户态就能实现多种设备文件，甚至是文件系统。接下来，我们就会借助这些机制，配合现代虚拟化技术，在Linux环境下把驱动全部开发好。这样做的好处是绕开了那个经典的死循环：写操作系统时，究竟是先有内核还是先有驱动？

用户态下开发比较方便，程序可以直接用gdb调试。我们不仅会在Linux下先把驱动写完，还会借助Syscall User Dispatch等现代Linux特性，把操作系统绝大部分内核代码也都在用户态里开发和调试好。

最后只需要惊险一跃，duang的一声，就把整个操作系统在QEMU里运行起来了。你会发现写操作系统竟如此容易。废话不多说，我们赶紧开始吧。
