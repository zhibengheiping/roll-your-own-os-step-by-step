# 开发环境

本教程使用Fedora作为开发环境。

若您使用的是Windows，请通过Windows Subsystem for Linux (WSL2)安装并运行官方的Fedora Linux 43。请先确保您的系统已正确安装并启用WSL2，具体方法可参考微软官方文档：[设置WSL开发环境](https://learn.microsoft.com/zh-cn/windows/wsl/setup/environment)。环境就绪后，以管理员身份打开PowerShell或Windows终端，执行以下命令即可完成Fedora 43的安装：

```powershell
wsl --install FedoraLinux-43
```

安装完成后，通过`wsl -d FedoraLinux-43`启动系统，按提示完成初始用户创建即可。

若您使用的是其他Linux发行版，可以使用Distrobox来安装和运行Fedora容器。[Distrobox](https://distrobox.it/)是一个基于容器技术的工具，允许您在任意Linux宿主系统中无缝创建其他发行版的环境，并自动共享宿主系统的HOME目录、网络、USB设备及图形界面。

首先在宿主机上安装Distrobox及容器引擎Podman。多数发行版可通过包管理器直接安装，例如在Ubuntu/Debian中执行`sudo apt install distrobox podman`，在Arch Linux中执行`sudo pacman -S distrobox podman`。随后执行以下命令创建Fedora 43容器环境：

```
distrobox create --name fedora-43 --image docker.io/fedora:43
```

创建完成后，使用`distrobox enter fedora-43`即可进入该容器。终端工具建议使用Ptyxis。Ptyxis是一款专为容器化工作流设计的GNOME终端模拟器，能够与Podman、Toolbox、Distrobox等工具深度集成，自动识别并快速切入已创建的容器环境。

若您使用的是Fedora Silverblue、Kinoite等不可变版本，系统通常已内置Toolbox和Ptyxis，无需额外安装Distrobox，您可以执行以下命令创建Fedora 43容器环境：

```
toolbox create -r 43
```

创建完成后，使用`toolbox enter -r 43`即可进入该容器。
